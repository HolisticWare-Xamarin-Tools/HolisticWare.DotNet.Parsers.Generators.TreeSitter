## Project Overview

This repo builds cross-platform native shared libraries for tree-sitter language parsers. The outputs are `.dylib` (macOS/iOS), `.so` (Linux/Android), and `.dll` (Windows) binaries consumed by .NET applications for code intelligence (IntelliSense, AI RAG/GraphRAG).

The build system uses **CMake + Zig cross-compilation** to produce binaries for 7 platforms from a single host machine.

## Workflow

### Step 1: Fetch Sources

```bash
./download-clone-native.sh
```

Downloads and unpacks tree-sitter core and all grammar sources into `externals/native/core/` and `externals/native/grammars/`.

### Step 2: Build Native Binaries

```bash
cd externals/native
./build-all.sh                                          # All platforms, all grammars
./build-all.sh --platform macos                         # Single platform
./build-all.sh --grammar-filter c-sharp,typescript,rust # Specific grammars
./build-all.sh --optimize                               # Parallel + size profile
./build-all.sh --profile size|speed|balanced|debug
./build-all.sh --clean                                  # Clean then build
./build-all.sh --verify-only                            # Verify existing binaries
```

Outputs land in `externals/native/out-<profile>/<platform>/<arch>/`.

### Step 3: Verify Binaries

```bash
cd externals/native
./verify-binaries.sh
./verify-binaries.sh --platform macos
```

### Sync CMakeLists to docs/build Mirror

```bash
./docs/build/scripts/backup.sh   # Copy modified CMakeLists.txt files to docs/build mirror
./docs/build/scripts/restore.sh  # Restore from docs/build mirror
```

## Architecture

### Directory Layout

```
externals/native/
  core/tree-sitter-master/      # tree-sitter core C library (fetched by download script)
  grammars/tree-sitter-<lang>-*/# Per-language grammar sources (fetched by download script)
  bindings/                     # C# binding repos (csharp-tree-sitter, dotnet-tree-sitter)
  toolchains/                   # CMake toolchain files (zig-*.cmake, ndk-*.cmake)
  out-<profile>/                # Build outputs by profile (balanced/size/speed/debug)
  build-<profile>/              # CMake build trees (gitignored)
  logs-<profile>/               # Build and verification logs

docs/build/
  externals/native/             # Mirror of externals/native for CI/alternative build context
  scripts/backup.sh             # Syncs CMakeLists.txt files from source to docs mirror
  scripts/restore.sh            # Restores CMakeLists.txt files from docs mirror to source

source/                         # .NET managed code (future .NET bindings/wrappers)
toolchains/                     # Root-level toolchains (symlinked or duplicate)
```

### Two-Phase Build (per platform)

1. **Core library**: builds `libtree-sitter.{ext}` from `core/tree-sitter-master/`
2. **Each grammar**: built individually from its own `CMakeLists.txt`, linking against the core

This avoids CMake duplicate-target conflicts when building many grammars in the same configure step.

### Toolchains

Zig is the cross-compiler for all non-Android targets. Each toolchain file in `externals/native/toolchains/` sets `CMAKE_C_COMPILER` to `zig cc` with the appropriate `-target` triple. Android uses the NDK LLVM toolchain (`ndk-aarch64-android.cmake`), generated by `setup-android.sh`.

### Supported Targets

| Platform | Arch   | Output | Toolchain                  |
|----------|--------|--------|----------------------------|
| macOS    | x86_64 | dylib  | zig-x86_64-macos.cmake     |
| macOS    | arm64  | dylib  | zig-aarch64-macos.cmake    |
| Linux    | x86_64 | so     | zig-x86_64-linux.cmake     |
| Linux    | arm64  | so     | zig-aarch64-linux.cmake    |
| Windows  | x64    | dll    | zig-x86_64-windows.cmake   |
| Android  | arm64  | so     | ndk-aarch64-android.cmake  |
| iOS      | arm64  | dylib  | zig-aarch64-ios.cmake      |

## Prerequisites

- **CMake 3.13+**
- **Zig** (`brew install zig` on macOS)
- **Standard tools**: `file`, `nm`, `objdump`
- **Android NDK r25c+** (only for Android builds; set `ANDROID_NDK_HOME` or `ANDROID_HOME`)

## CI

GitHub Actions workflow (`.github/workflows/build-treesitter.yml`) runs matrix builds across all 7 platforms on push to `main`/`develop` when files under `externals/native/` change, and weekly on Monday.

## Key Files

- `externals/native/build-all.sh` — main build orchestrator
- `externals/native/verify-binaries.sh` — binary verification
- `externals/native/setup-android.sh` — generates NDK toolchain file
- `externals/native/benchmark.sh` — performance benchmarking
- `externals/native/patch-grammars-for-opt.sh` — patches grammar CMakeLists for size/speed optimization
- `externals/native/profiles.md` — detailed binary size/performance profiles per grammar/platform
